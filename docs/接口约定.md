# 前后端接口约定

## 通用规则

- 前端只调用本文件中的业务 API，不直接连接 RunningHub。
- JSON 请求使用 `Content-Type: application/json`；上传由浏览器自动生成 multipart boundary。
- 前端始终携带 `credentials: include`，后端可使用 HTTP-only Cookie Session。
- 所有时间为 ISO 8601 字符串。
- 通用失败响应：

```json
{
  "error": {
    "code": "GENERATION_FAILED",
    "message": "动画生成失败，请重试。"
  }
}
```

后端不得向前端返回 API Key、Workflow Secret、堆栈、本地绝对路径或 RunningHub 内部错误详情。

## 统一类型

### JobMode

```ts
type JobMode = "initial" | "text_edit" | "motion_edit" | "combined_edit";
```

### JobStatus

```ts
type JobStatus =
  | "PENDING" | "SUBMITTING" | "QUEUED" | "RUNNING"
  | "PROCESSING_OUTPUT" | "SUCCEEDED" | "FAILED"
  | "CANCELLING" | "CANCELLED";
```

### GestureData

```ts
interface GesturePoint { x: number; y: number; t: number }
interface HandGesture { pathPoints: GesturePoint[]; speedProfile?: number[] }
interface GestureData {
  leftHand?: HandGesture;
  rightHand?: HandGesture;
  durationMs?: number;
}
```

`x`、`y` 为建议范围 0–1 的归一化坐标；`t` 是从录制开始计算的毫秒时间。

### CreateJobInput

```ts
interface CreateJobInput {
  clientRequestId: string;
  projectId: string;
  mode: JobMode;
  brief: string;
  target?: string;
  instruction?: string;
  gesture?: GestureData;
  referenceAssetId?: string;
  baseVersionId?: string | null;
  settings?: { seed?: number; steps?: number; durationSeconds?: number };
}
```

### Job

```ts
interface Job {
  id: string;
  mode: JobMode;
  status: JobStatus;
  queuePosition?: number | null;
  error?: { code?: string; message: string } | null;
  createdAt: string;
  completedAt?: string | null;
}
```

### MotionVersion

```ts
interface MotionVersion {
  id: string;
  projectId: string;
  sourceJobId: string;
  parentVersionId?: string | null;
  title: string;
  sourceType: JobMode;
  target?: string;
  brief?: string;
  instruction?: string;
  outputUrl: string;
  outputType: "video" | "image" | "gif" | "json" | "unknown";
  mimeType?: string;
  fileName?: string;
  createdAt: string;
}
```

### UploadedAsset

```ts
interface UploadedAsset {
  id: string;
  projectId: string;
  originalName: string;
  mimeType: string;
  previewUrl?: string;
}
```

## 1. 开始 Session

用途：按参与者编号创建或恢复研究身份和默认项目。前端使用位置：首次进入的 Participant Code 页面。

- 方法：`POST`
- 路径：`/api/session/start`
- 请求字段：`participantCode: string`，必填。

```json
{ "participantCode": "P01" }
```

成功响应：

```json
{
  "participant": { "id": "participant_001", "code": "P01" },
  "project": { "id": "project_001", "title": "Study Project" }
}
```

失败示例（HTTP 400）：

```json
{ "error": { "code": "INVALID_PARTICIPANT_CODE", "message": "参与者编号格式不正确。" } }
```

## 2. 恢复当前 Session

用途：页面刷新时恢复身份与项目。前端使用位置：`useParticipantSession`。

- 方法：`GET`
- 路径：`/api/session/current`
- 请求字段：无；身份来自 Cookie。

成功响应与“开始 Session”一致。

失败示例（HTTP 401）：

```json
{ "error": { "code": "SESSION_NOT_FOUND", "message": "当前会话不存在，请重新输入参与者编号。" } }
```

## 3. 上传资源

用途：上传 Reference，任务 JSON 仅引用返回的 `asset.id`。前端使用位置：用户提交任务前的上传阶段。

- 方法：`POST`
- 路径：`/api/assets`
- 类型：`multipart/form-data`
- 字段：`file`（必填文件）、`projectId`（必填）、`assetType`（`reference | gesture | other`）。

表单示意：

```text
file=<binary>
projectId=project_001
assetType=reference
```

成功响应：

```json
{
  "asset": {
    "id": "asset_001",
    "projectId": "project_001",
    "originalName": "reference.png",
    "mimeType": "image/png",
    "previewUrl": "/media/assets/asset_001"
  }
}
```

失败示例（HTTP 413）：

```json
{ "error": { "code": "FILE_TOO_LARGE", "message": "文件超过上传限制。" } }
```

## 4. 创建任务

用途：创建 Initial 或编辑任务；响应必须快速返回，不等待生成完成。前端使用位置：Generate Initial Motion 和 Apply Edit & Generate。

- 方法：`POST`
- 路径：`/api/jobs`
- 请求体：`CreateJobInput`
- `clientRequestId` 用于网络重试幂等；编辑模式必须提供 `baseVersionId`。

```json
{
  "clientRequestId": "request_abc123",
  "projectId": "project_001",
  "mode": "motion_edit",
  "brief": "A leaf falls slowly from the upper-left.",
  "instruction": "Accelerate near the end.",
  "gesture": {
    "leftHand": { "pathPoints": [{ "x": 0.2, "y": 0.8, "t": 0 }, { "x": 0.7, "y": 0.2, "t": 1800 }] },
    "durationMs": 1800
  },
  "referenceAssetId": "asset_001",
  "baseVersionId": "version_001"
}
```

成功响应（HTTP 202 或 200）：

```json
{
  "job": {
    "id": "job_001", "mode": "motion_edit", "status": "PENDING",
    "queuePosition": 1, "error": null,
    "createdAt": "2026-07-13T10:00:00Z", "completedAt": null
  }
}
```

失败示例（HTTP 409）：

```json
{ "error": { "code": "ACTIVE_JOB_EXISTS", "message": "当前已有生成任务，请等待或取消。" } }
```

## 5. 查询任务

用途：轮询状态并在成功时取得版本。前端使用位置：`useJobPolling`。

- 方法：`GET`
- 路径：`/api/jobs/:jobId`
- 路径字段：URL 编码后的 `jobId`。

处理中响应：

```json
{
  "job": {
    "id": "job_001", "mode": "initial", "status": "RUNNING",
    "queuePosition": null, "error": null,
    "createdAt": "2026-07-13T10:00:00Z", "completedAt": null
  },
  "version": null
}
```

成功响应：

```json
{
  "job": {
    "id": "job_001", "mode": "initial", "status": "SUCCEEDED",
    "queuePosition": null, "error": null,
    "createdAt": "2026-07-13T10:00:00Z", "completedAt": "2026-07-13T10:02:00Z"
  },
  "version": {
    "id": "version_001", "projectId": "project_001", "sourceJobId": "job_001",
    "parentVersionId": null, "title": "Initial Motion", "sourceType": "initial",
    "brief": "A leaf falls slowly.", "outputUrl": "/media/outputs/version_001.mp4",
    "outputType": "video", "mimeType": "video/mp4", "fileName": "initial-motion.mp4",
    "createdAt": "2026-07-13T10:02:00Z"
  }
}
```

生成失败仍返回任务响应：

```json
{
  "job": {
    "id": "job_001", "mode": "initial", "status": "FAILED",
    "error": { "code": "GENERATION_FAILED", "message": "动画生成失败，请重试。" },
    "createdAt": "2026-07-13T10:00:00Z", "completedAt": "2026-07-13T10:02:00Z"
  },
  "version": null
}
```

无权访问或不存在时返回 HTTP 404/403 和通用错误格式。

## 6. 取消任务

用途：取消尚未结束的任务。前端使用位置：任务状态条 Cancel 按钮。

- 方法：`POST`
- 路径：`/api/jobs/:jobId/cancel`
- 请求字段：无。

成功响应：

```json
{
  "job": {
    "id": "job_001", "mode": "initial", "status": "CANCELLED",
    "queuePosition": null, "error": null,
    "createdAt": "2026-07-13T10:00:00Z", "completedAt": "2026-07-13T10:00:10Z"
  }
}
```

失败示例（HTTP 409）：

```json
{ "error": { "code": "JOB_ALREADY_FINISHED", "message": "任务已经结束，无法取消。" } }
```

## 7. 获取项目版本

用途：读取长期版本事实来源，按创建时间从新到旧返回。前端使用位置：`useProjectVersions`、Timeline 和 Version History。

- 方法：`GET`
- 路径：`/api/projects/:projectId/versions`
- 路径字段：URL 编码后的 `projectId`。

成功响应：

```json
{
  "versions": [
    {
      "id": "version_001", "projectId": "project_001", "sourceJobId": "job_001",
      "parentVersionId": null, "title": "Initial Motion", "sourceType": "initial",
      "outputUrl": "/media/outputs/version_001.mp4", "outputType": "video",
      "mimeType": "video/mp4", "fileName": "initial-motion.mp4",
      "createdAt": "2026-07-13T10:02:00Z"
    }
  ]
}
```

失败示例（HTTP 403）：

```json
{ "error": { "code": "PROJECT_FORBIDDEN", "message": "无权访问该项目。" } }
```

## 8. 记录研究事件

用途：记录版本选择和导出等非敏感交互。前端使用位置：版本点击和导出；失败不得阻塞主流程。

- 方法：`POST`
- 路径：`/api/events`
- 请求字段：`projectId`、`eventType`，可选 `eventData`。

```json
{
  "projectId": "project_001",
  "eventType": "version_selected",
  "eventData": { "versionId": "version_001" }
}
```

成功响应：

```json
{ "ok": true }
```

失败示例（HTTP 400）：

```json
{ "error": { "code": "INVALID_EVENT", "message": "事件格式不正确。" } }
```

## 相对资源 URL

真实模式下，若 `previewUrl` 或 `outputUrl` 是以 `/` 开头的相对地址，前端会使用 `VITE_API_BASE_URL` 作为根地址；完整 HTTPS URL 保持不变。后端应保证 Cookie、CORS、媒体 Range 请求和下载响应头与部署方式兼容。
