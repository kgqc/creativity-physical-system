# 前端交接说明

## 交付范围

当前前端负责人交付：

- 完整 React + Vite + TypeScript 源代码；
- 可独立运行的 Mock 模式；
- 统一 `ApiService`、真实 API Adapter 和 Mock API Adapter；
- 稳定的任务、版本、Session、上传和 Gesture 类型；
- 任务状态、错误、重试、取消和连接提示界面；
- Reference 上传预览、Initial Motion、Text Edit、Motion Edit、Combined Edit；
- 服务端版本读取、版本选择和基于旧版本继续编辑；
- 示例视频、动画下载和项目 JSON 导出；
- `dist/` 构建能力与中文交接文档。

## 合作者职责

合作者负责：

- 实现 `docs/接口约定.md` 中的 HTTP API；
- 在后端安全保存 RunningHub API Key 并连接 Workflow；
- 将上传资源转存到 RunningHub 或自己的存储；
- 创建、查询和取消任务；
- 持久化 Session、项目、任务、事件和版本；
- 校验用户身份并实现多用户数据隔离；
- 返回浏览器可播放或可下载的结果 URL；
- 提供公共访问地址并托管 `dist/`，或正确配置跨域 Cookie/CORS；
- 把 RunningHub 错误转换成统一前端错误格式。

前端请求中不会包含 RunningHub Workflow ID、节点 ID、API Key、Webhook 或内部 Task ID。

## 推荐接入顺序

1. 运行 Mock 模式，完整体验用户流程。
2. 实现返回固定数据的测试后端。
3. 设置 `VITE_USE_MOCK_API=false`，验证 Session 恢复。
4. 验证 Reference 上传与 `assetId`。
5. 验证创建任务和 `clientRequestId` 幂等。
6. 验证状态轮询、网络暂时失败和取消任务。
7. 验证成功结果播放、下载及版本刷新。
8. 再连接真实 RunningHub。
9. 验证 Text Edit 的 `baseVersionId` 和 `instruction`。
10. 验证 Motion Edit / Combined Edit 的 `GestureData`。
11. 验证多用户隔离、跨域 Cookie 和多终端访问。

## 构建交付

```bash
npm install
npm run build
```

合作者可托管 `dist/`，也可在自己的部署流水线中重新构建。环境变量是构建期配置，改变真实 API 地址后需要重新构建。
